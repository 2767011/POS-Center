# Memory Index

Последняя оптимизация: 2026-06-03

## Проект

- POS Center — набор Windows-утилит для управления и обновления прошивки ККТ POS Center / Штрих-М через COM-драйвер `AddIn.DrvFR`.
- Основные точки входа: `run.bat` (интерактивное меню), `auto_update.bat` (удалённый автоцикл), `kkt_firmware_update.py` (прошивка), `kkt_info.py` (диагностика), `kkt_dump_tables.py` (дамп таблиц), `probe_com.py` (инспекция COM).
- Общий Python-модуль: `kkt_driver.py`; профильная справка для агента: `AgentSkill_KKT.md`; пользовательская карта проекта: `ONBOARDING.md`.

## Инфраструктура

- Проект ориентирован на Windows, portable Python, `pywin32`, BAT/PowerShell-обёртки и зарегистрированный COM-драйвер DrvFR соответствующей разрядности.
- В проекте присутствуют архивы portable Python, драйверов VCOM/DFU, TSPIoT-пакет и PDF-руководства.

## Ограничения

- Не хранить секреты, персональные данные, внутренние хосты и доступы в памяти.
- В `ONBOARDING.md` и `server_snapshot/` есть чувствительные серверные сведения; не переносить их значения в память и не цитировать без необходимости.
- `DrvFR.dll` должен совпадать по разрядности с Python; COM Surrogate для этого драйвера не считается рабочим решением.

## Факты

- Локальная память проекта инициализирована 2026-06-03.
- `kkt_firmware_update.py` делает бэкап таблиц, проверяет/закрывает смену, выбирает прошивку по варианту ККТ, запускает DFU-обновление и пишет лог/JSON-отчёт.
- `auto_update.bat` требует прав администратора, работает из временной KKT-директории, скачивает компоненты, проверяет Python/pywin32/COM-драйвер и запускает прошивку.
- Перед запуском прошивки `auto_update.bat` скачивает `VCOM+DFU.zip`, распаковывает USB VCOM/DFU-драйверы и пытается установить их через `install_dfu_driver.bat`/`pnputil`; ошибка установки не должна прерывать прошивку, только выводить крупное предупреждение в конце.
