# POS Center KKT Management Toolkit

Набор Windows-скриптов для диагностики, резервного копирования таблиц и обновления прошивки ККТ POS Center / Штрих-М через COM-драйвер `AddIn.DrvFR`.

Проект рассчитан на локальный запуск на кассе или удалённый запуск через PsExec. Основной сценарий: подготовить окружение, проверить драйверы, сохранить таблицы ККТ, выбрать подходящую прошивку и выполнить DFU-обновление.

## Что внутри

| Файл | Назначение |
|------|------------|
| `run.bat` | Интерактивное меню для локального запуска диагностики, дампа, прошивки и проверки COM-драйвера |
| `auto_update.bat` | Полный удалённый цикл обновления: скачивание файлов, проверка окружения, драйверы, прошивка |
| `download.ps1` | Скачивание updater-скриптов, прошивок и пакета `VCOM+DFU.zip` с внутреннего веб-сервера |
| `install_python.ps1` | Установка portable Python с `pywin32`; Windows 7 получает отдельный пакет Python 3.8 |
| `install_dfu_driver.bat` | Добавление подписантов `.cat` в `TrustedPublisher` и установка USB VCOM/DFU-драйверов через `pnputil` |
| `register_drvfr.bat` | Регистрация COM-драйвера `DrvFR.dll` подходящей разрядности |
| `kkt_driver.py` | Общие функции: создание COM-объекта, TCP/COM-подключение, чтение свойств и таблиц |
| `kkt_info.py` | Диагностика ККТ: модель, серийный номер, версия ПО, ФН, ОФД, сеть |
| `kkt_dump_tables.py` | Дамп всех или выбранных таблиц ККТ в CSV |
| `kkt_firmware_update.py` | Основной сценарий прошивки: precheck, бэкап таблиц, выбор `.bin`, DFU, реконнект |
| `probe_com.py` | Инспекция свойств и методов COM-драйвера |

Дополнительные материалы:

- `ONBOARDING.md` — подробная внутренняя карта проекта и операционные заметки.
- `AgentSkill_KKT.md` — справка по COM-драйверу и типовым операциям.
- `manual/` — PDF-документация и примеры.
- `VCOM+DFU.zip` / `VCOM+DFU/` — пакет USB VCOM/DFU-драйверов.

## Быстрый старт

Локально на машине с подключённой ККТ:

```bat
setup.bat
run.bat
```

Прямой запуск обновления из локальной папки:

```bat
run_update.bat
```

Удалённый автоматический запуск через PsExec:

```bat
PsExec \\KASSA -s -c auto_update.bat
```

Перед запуском проверьте `config.bat`: там задаются IP/порт ККТ и путь к прошивкам по умолчанию.

## Сценарий автообновления

`auto_update.bat` выполняет шаги в таком порядке:

1. Скачивает актуальные скрипты, прошивки и `VCOM+DFU.zip`.
2. Проверяет или устанавливает portable Python.
3. Проверяет `pywin32`.
4. Проверяет или регистрирует COM-драйвер `AddIn.DrvFR`.
5. Пытается установить USB VCOM/DFU-драйверы.
6. Проверяет наличие `.bin` прошивок.
7. Запускает `kkt_firmware_update.py --force`.

Ошибка установки USB VCOM/DFU-драйверов не останавливает прошивку. Она пишется в `dfu_driver_install.log`, а в конце вывода показывается крупное предупреждение.

## Прошивка ККТ

`kkt_firmware_update.py` делает следующее:

1. Подключается к ККТ через COM-драйвер.
2. Получает текущую версию ПО.
3. Проверяет состояние смены и закрывает её при необходимости.
4. Определяет вариант ККТ по таблице 23, строке 1, полю 11.
5. Выбирает подходящую прошивку из папки с `.bin`.
6. Создаёт CSV-бэкап всех таблиц.
7. Запускает DFU-обновление через драйвер.
8. Ожидает завершения, переподключается и сверяет новую версию.
9. Применяет пост-настройки, если они нужны для выбранной прошивки.

Пример ручного запуска:

```bat
python kkt_firmware_update.py --ip 192.168.137.111 --port 7778 --file firmware --force
```

## USB VCOM/DFU-драйверы

`install_dfu_driver.bat` устанавливает два INF из `VCOM+DFU`:

- `Windows\INF\dfu\lpc-composite89-dfu.inf`
- `Windows\INF\vcom\lpc-ucom-vcom.inf`

Перед вызовом `pnputil` скрипт берёт подписанта соседнего `.cat` через `Get-AuthenticodeSignature` и добавляет сертификат в `LocalMachine\TrustedPublisher`. Это нужно для non-interactive запуска: Windows не может показать окно подтверждения доверия издателю при запуске через PsExec или scheduled task.

Ручной запуск:

```bat
install_dfu_driver.bat VCOM+DFU
```

## Структура веб-сервера

Веб-сервер используется как файловый источник для удалённого автообновления. Публичный корень Apache настроен на каталог с файлами, а `download.ps1` забирает updater-скрипты из раздела `KKT/Updater` и прошивки из `KKT/FW_FR`.

Ожидаемая структура публичного каталога:

```text
/var/www/files/
├── KKT/
│   ├── Updater/
│   │   ├── auto_update.bat
│   │   ├── download.ps1
│   │   ├── install_python.ps1
│   │   ├── install_dfu_driver.bat
│   │   ├── register_drvfr.bat
│   │   ├── setup.bat
│   │   ├── run.bat
│   │   ├── run_update.bat
│   │   ├── config.bat
│   │   ├── kkt_driver.py
│   │   ├── kkt_firmware_update.py
│   │   ├── kkt_dump_tables.py
│   │   ├── kkt_info.py
│   │   ├── probe_com.py
│   │   └── VCOM+DFU.zip
│   └── FW_FR/
│       ├── *.bin
│       └── table_after_update.csv
├── admin.php
├── .admin_password
├── .settings.json
├── .lang/
└── .trash/
```

Назначение разделов:

- `KKT/Updater/` — скрипты автообновления, общие Python-модули, BAT/PowerShell-обёртки и архив USB VCOM/DFU-драйверов.
- `KKT/FW_FR/` — прошивки ККТ (`*.bin`) и дополнительная таблица пост-настроек `table_after_update.csv`.
- `admin.php` — файловый менеджер веб-раздела; доступ к нему должен быть закрыт паролем.
- `.admin_password`, `.settings.json`, `.lang/`, `.trash/` — служебные файлы файлового менеджера; они не должны отдаваться как публичные файлы.

Минимальная идея Apache vhost:

```apache
DocumentRoot /var/www/files
Alias /admin /var/www/files/admin.php
IndexIgnore .admin_password admin.php .lang .trash .settings.json
```

В публичной документации не фиксируйте реальные IP-адреса, пароли, хэши и административные URL. Для рабочих значений используйте локальные операционные заметки или переменные в скриптах.

## Требования

- Windows с правами администратора.
- COM-драйвер `AddIn.DrvFR` (`DrvFR.dll`) той же разрядности, что и Python.
- Python с `pywin32`; в автоустановке используется portable Python.
- Доступ к ККТ по TCP или COM.
- Для удалённого сценария: PsExec и доступ к административному запуску на кассе.

## Важные особенности

- COM Surrogate для `DrvFR.dll` не используется: драйвер должен быть нативной подходящей разрядности.
- При запуске через PsExec рабочая папка может оказаться не там, где лежит `.bat`; поэтому автообновление работает из временной KKT-директории.
- Для совместимости со старыми Windows ZIP распаковывается без обязательной зависимости от `Expand-Archive`.
- Логи прошивки пишутся рядом со скриптами в рабочей папке автообновления.
- Секреты, пароли и внутренние административные URL не должны попадать в публичные README/логи.

## Полезные команды

Диагностика:

```bat
python kkt_info.py --ip 192.168.137.111 --port 7778
```

Дамп таблиц:

```bat
python kkt_dump_tables.py --ip 192.168.137.111 --port 7778 --output tables_dump.csv
```

Инспекция COM-драйвера:

```bat
python probe_com.py
```

## Где искать логи

При автообновлении основные файлы обычно находятся в рабочей KKT-директории:

- `update_kkt.log` — ход прошивки.
- `update_report.json` — машинно-читаемый итог.
- `dfu_driver_install.log` — установка USB VCOM/DFU-драйверов.
- `tables_backup_<serial>_<timestamp>.csv` — бэкап таблиц перед прошивкой.
